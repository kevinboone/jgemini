# JGemini project status

April 2026

JGemini is basically functional, and it's my main interface to the
Gemini/Gopher world these days. I use it every day. It's reasonably stable,
although there remains a certain (unreproducible) tendency to hang on exit --
not a problem if you're running from a command line. There are still a few
(unreproducible) occasions on which JGemini will spit out an incomprehensible
stack back-trace, although they don't all cause problems in operation.

JGemini is deliberately simplistic and old-fashioned in its appearance and
operation -- I don't consider this a defect.  The lack of tabbed browsing
support, for example, isn't something I plan to fix. I don't mind if somebody
else wants to add it, of course ;) 

JGemini is feature-poor compared to the other, better-established Gemini
clients. Some of the missing functionality I plan to add in due course; some I
don't. JGemini is, perhaps, a little ahead of other clients in that it has full
support for Markdown. The built-in documentation, for example, is all written
in Markdown, which I think makes it look nicer, as well as improving ease of
navigation. Not everybody agrees, however.

My most pressing concerns right now are the lack of any kind of support for
streaming, and the need for a better way to search URL history.
